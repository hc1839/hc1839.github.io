Detailed Description

template<typename T>
class tanuki::parallel::grs::Actuator< T >

Actuator of the Givens Relaxation Scheme (GRS).

GRS is a generalization of the Jacobi Relaxation Scheme (JRS) developed by Rajasekaran 2008.

Template Parameters
TType of elements in an Armadillo matrix.

Parameters
mpi_commMPI communicator. The level of thread support must be at least MPI_THREAD_FUNNELED. Number of MPI processes at each host must be the same.
max_threadsMaximum number of threads to use. It cannot exceed omp_get_max_threads().
sidednessSidedness of GRS to perform. If a two-sided GRS is to be performed, matrix must be a square matrix (otherwise std::invalid_argument is thrown), and only columns will be queried for rotations; the rotations are then applied to both rows and columns.
init_relaxInitial relaxation parameter. If it is not in the range \( [0, 1) \), std::domain_error is thrown.
num_groupsPositive number of groups of rotation sets to form. If zero, std::domain_error is thrown.
max_iterationsMaximum number of iterations. If not positive, std::domain_error is thrown.
relax_fnFunction that returns the relaxation parameter to use for the next group of rotation sets. Arguments are iteration index, previous relaxation parameter, and current group index, respectively. If the next relaxation parameter is not in the range \( [0, 1) \), std::domain_error is thrown.
convergence_checkerFunction that returns whether convergence is attained given the previous and current matrices, respectively, from the transformation process.

Actuates GRS.

The rotation returned from the inquiry function is not relaxed. Relaxation is performed by the actuator. Matrix index pair is always MatrixIndexPair::PairType::COLUMNS.

All MPI processes must invoke this outside any OpenMP parallel region.
